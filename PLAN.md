# Gorchev-V — План реализации

## Стек
- Next.js + TypeScript
- Tailwind CSS + shadcn/ui
- PostgreSQL + Prisma ORM
- Фото — локальное хранение

## Критерии Go/No-Go

### Продолжаем если:
- [ ] Кладовщик ведёт учёт через систему без помощи
- [ ] Рабочие пользуются терминалом каждую смену
- [ ] Директор перестала спрашивать "а сколько у нас на складе" — смотрит сама

### Меняем курс если:
- [ ] Кладовщик возвращается к тетрадке — интерфейс слишком сложный
- [ ] Рабочие не пользуются терминалом — неудобно или медленно

## После MVP

### Ближайшее (высокий приоритет)
- Голосовой ввод для кладовщика
- Telegram-бот для директора
- Спецификации (BOM): деталь → материалы/сырьё (лист стали, кол-во). Расчёт потребности в материалах для производства
- Инвентаризация: кладовщик сверяет остатки (факт vs учёт), система показывает расхождения
- Учёт брака (?) — рабочий отмечает брак при вводе количества

### Среднесрочное
- Дашборд с аналитикой
- Себестоимость: материалы (из спецификаций) + зарплата (сдельная) = себестоимость детали/изделия
- Агент закупок: автоматическая рассылка запросов цен поставщикам на email, сбор и сравнение предложений
- Интеграция с 1С
- Тиражирование на другие цеха
