# Gorchev-V — План реализации

## Стек
- Next.js + TypeScript
- Tailwind CSS + shadcn/ui
- PostgreSQL + Prisma ORM
- Фото — локальное хранение

## Порядок реализации

### Фаза 1: Фундамент
- [ ] Инициализация проекта (Next.js, Prisma, shadcn/ui)
- [ ] Схема БД: пользователи, роли, PIN-коды
- [ ] Схема БД: каталог (категории, изделия, комплектующие, фото)
- [ ] Схема БД: склад (остатки, приход/расход)
- [ ] Схема БД: выработка (кто, что, сколько, когда)
- [ ] Схема БД: зарплата (расценки за деталь)
- [ ] Авторизация по PIN-коду
- [ ] Найти в интернете фото штампованных металлоизделий, комплектующих, категорий (для демо-каталога)
- [ ] Составить названия категорий и позиций каталога на основе реальной номенклатуры цеха штамповки
- [ ] Заполнение демо-данными (штампованные изделия, фото, комплектующие)

### Фаза 2: Каталог и склад
- [ ] Древовидный каталог изделий (категории → изделия → комплектующие)
- [ ] Карточка изделия: галерея фото, описание, список комплектующих
- [ ] Управление каталогом (добавить/удалить/редактировать позиции, загрузка фото)
- [ ] Склад: приход/расход через простой интерфейс
- [ ] Склад: отображение остатков в дереве каталога
- [ ] Расчёт "сколько изделий можно собрать" и "что нужно докупить"

### Фаза 3: Терминал рабочего
- [ ] Экран ввода PIN (большие кнопки, как банкомат)
- [ ] Выбор изделия/комплектующей с фото
- [ ] Просмотр фотогалереи детали
- [ ] Ввод количества + кнопка "Отправить"
- [ ] Автовыход по таймауту бездействия (настраиваемый)
- [ ] История выработки рабочего в личном кабинете

### Фаза 4: Зарплата и админка
- [ ] Расчёт сдельной зарплаты (расценка за деталь x количество)
- [ ] Просмотр зарплаты за смену/неделю/месяц
- [ ] Админка директора: обзор склада, выработка по рабочим, зарплаты
- [ ] Управление пользователями (добавить/удалить рабочего, сменить PIN)

### Фаза 5: Запуск
- [ ] Тестирование на реальных данных клиента
- [ ] Деплой на сервер
- [ ] Обучение кладовщика и рабочих

## Критерии Go/No-Go

### Продолжаем если:
- [ ] Кладовщик ведёт учёт через систему без помощи
- [ ] Рабочие пользуются терминалом каждую смену
- [ ] Директор перестала спрашивать "а сколько у нас на складе" — смотрит сама

### Меняем курс если:
- [ ] Кладовщик возвращается к тетрадке — интерфейс слишком сложный
- [ ] Рабочие не пользуются терминалом — неудобно или медленно
