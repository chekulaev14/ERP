# Визуальный конструктор BOM-цепочек — план внедрения

Замена текущего wizard-конструктора на визуальный редактор цепочек: блоки позиций + стрелки с quantity. Пользователь мыслит маршрутами изготовления, не абстрактным графом.

---

## Ключевые решения

- BOM — глобальный граф зависимостей, не "приватная схема product". Каждая связь parent-child общая для всей системы.
- Конструктор работает только с существующей номенклатурой. Создание позиций — отдельный этап через Номенклатуру.
- Без drag & drop, без React Flow. CSS Flex + SVG-стрелки.
- Сохранение = upsert: связь есть → обновить quantity, нет → создать.
- Удаление связи — с проверкой использования в других product.

## Существующий API (готов к использованию)

- `GET /api/bom` — все BOM-связи
- `POST /api/bom` — создание/upsert связи (parentId, childId, quantity). Проверка циклов встроена.
- `PUT /api/bom` — обновление quantity
- `DELETE /api/bom` — удаление связи (parentId, childId)
- `GET /api/nomenclature` — список всех позиций

## Что нужно добавить на бэкенде

- `GET /api/bom/tree?itemId=X` — рекурсивный обход графа вниз от позиции. Возвращает полное дерево зависимостей для загрузки в конструктор (фаза 3).

## Фаза 1: Линейная ветка (MVP)

Пользователь выбирает product и строит одну цепочку: [Сырьё] → [Заготовка] → ... → [Изделие].

### UI

Верхняя часть:
- Выбор конечного product (SearchableSelect, фильтр type=product)
- Кнопка "Сохранить"

Центральная область:
- Горизонтальная строка блоков слева направо
- Каждый блок: выбор позиции (SearchableSelect), отображает название + код + тип + единицу
- Между блоками: стрелка + поле quantity
- Кнопка "+" справа от последнего блока для добавления шага
- Кнопка удаления на каждом блоке
- Финальный блок справа — выбранный product (фиксированный, не редактируется)

### Сохранение

Цепочка [A] →(qty1) [B] →(qty2) [C] →(qty3) [Product] превращается в:
- POST /api/bom { parentId: B, childId: A, quantity: qty1 }
- POST /api/bom { parentId: C, childId: B, quantity: qty2 }
- POST /api/bom { parentId: Product, childId: C, quantity: qty3 }

API уже делает upsert — дубли невозможны.

### Состояние UI

```
selectedProductId: string | null
chain: Array<{ itemId: string | null, quantity: number }>
```

### Компоненты

- `ChainConstructor` — оркестратор (выбор product, управление состоянием, сохранение)
- `ChainBlock` — презентационный блок позиции (SearchableSelect + инфо)
- `ChainArrow` — стрелка с полем quantity

Файлы: app/src/components/warehouse/constructor/ (новая папка, заменяет текущую)

Критерий готовности: можно выбрать product, построить цепочку из 3-4 блоков, сохранить — BOM-связи создаются в БД.

## Фаза 2: Несколько веток

Пользователь добавляет параллельные цепочки, которые сходятся в один product.

### UI

- Кнопка "Добавить ветку" под существующими ветками
- Каждая ветка — отдельная горизонтальная строка
- Все ветки справа визуально сходятся в блок product
- SVG-линии от последнего блока каждой ветки к product
- Кнопка удаления ветки

### Состояние UI

```
selectedProductId: string | null
branches: Array<{
  id: string
  chain: Array<{ itemId: string | null, quantity: number }>
  quantityToProduct: number
}>
```

### Визуальная схема

```
[Лист 4мм] →(3кг) [Пластина] →(1) [Крюк гнутый] →(1)─┐
                                                         ├→ [Кронштейн]
[Пруток 8мм] →(0.5кг) [Ось] →(2)──────────────────────┘
```

Файлы: те же компоненты + BranchRow, ProductNode

Критерий готовности: можно создать product с двумя ветками, сохранить — все BOM-связи создаются корректно.

## Фаза 3: Загрузка существующего BOM

При выборе product, у которого уже есть BOM — конструктор загружает и отображает существующие цепочки.

### Бэкенд

Новый endpoint: `GET /api/bom/tree?itemId=X`
- Рекурсивный обход от itemId вниз по child
- Возвращает дерево: { item, quantity, children: [...] }

### Фронтенд

- Преобразование дерева → ветки (алгоритм: DFS, каждый путь от leaf до root = ветка)
- Отображение существующих связей в конструкторе
- Редактирование quantity → PUT /api/bom
- Добавление новых блоков/веток → POST /api/bom
- Удаление блока: отвязка от текущей ветки (не удаляет глобальную связь автоматически)

### Удаление связи

Отдельное действие с подтверждением. Перед удалением — запрос: "какие ещё product используют эту связь?". Если используется — предупреждение.

Файлы: bom.service.ts (getBomTree), api/bom/tree/route.ts, ChainConstructor (загрузка)

Критерий готовности: при выборе product с существующим BOM — цепочки отображаются, можно редактировать и дополнять.

---

## Фаза 4: Валидации и UX-полировка

- Ограничение типов: первый блок ветки = material или blank, промежуточные = blank, финальный = product
- Предупреждение при пустых блоках или quantity ≤ 0
- Подсветка блоков по типу (цвета из typeColors)
- Отображение "эта связь уже используется в X изделиях" при наведении
- Toast с итогом после сохранения: "Создано N связей, обновлено M"

## Порядок внедрения

Фаза 0 → Фаза 1 → тест на реальных данных → Фаза 2 → Фаза 3 → Фаза 4

Каждая фаза = отдельный коммит. После фазы 1 конструктор уже функционален для простых изделий (одна цепочка).
