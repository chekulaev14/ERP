# Gorchev-V — Продуктовая стратегия

## Видение

Самый простой производственный софт для малых и средних производств в РФ. Клиент настраивает систему за 2 дня без интегратора, без обучения, без документации. AI делает всю рутину, живой колл-центр страхует.

---

## Текущий статус: пилот

Один клиент — металлические двери. Обкатка на реальном цехе. Работающие модули:
- Терминал рабочего (PIN, каталог, ввод выработки)
- Номенклатура (93 позиции, BOM, конструктор изделия)
- Склад (остатки, операции, сборка по BOM)
- Производственные заказы (создание, запуск, завершение)
- Auth (PIN для рабочих, email/password для управления)

---

## Целевая аудитория

**Сегмент:** малые и средние производства (5–100 человек), металлообработка, мебель, пластик, пищёвка.

**Портрет клиента:** директор/собственник, который:
- Ведёт учёт в тетрадке, Excel или голове
- Пробовал 1С — дорого, сложно, нужен интегратор
- Не готов тратить месяцы на внедрение
- Хочет видеть остатки, выработку и зарплату "здесь и сейчас"

## Конкурентное преимущество

### Почему не 1С, VOGBIT, Ruli24

| | 1С | VOGBIT | Gorchev-V |
|---|---|---|---|
| Внедрение | 1–6 мес, интегратор | 2–4 нед, обучение | **2 дня, сам** |
| Стоимость старта | 100–500K | 50–200K | **0 ₽** |
| Сложность | Высокая | Средняя | **Минимальная** |
| Подходит для малых | Нет | Частично | **Да** |

### Ключевой барьер конкурентов

Чтобы повторить — нужны данные сотен реальных производств (BOM, нормы расхода, процессы по отраслям). Каждый новый клиент делает продукт умнее. Network effect через данные.

## Воронка клиента

```
Этап 1: Привлечение
  Сайт / рекомендации / реклама
       ↓
Этап 2: AI-онбординг (бесплатно, самостоятельно)
  "Чем занимаетесь?" → Отрасль определена
  AI предлагает шаблоны от похожих производств
       ↓
Этап 3: Ввод данных (AI делает за клиента)
  📎 Excel → AI парсит любой формат
  🎤 Голос → Speech-to-text → AI структурирует
  💬 Текст → AI создаёт номенклатуру
  📷 Фото → AI распознаёт
  AI знает сленг ("тройка", "сороковка", "полтинник")
  AI подсказывает: "Утеплитель забыли?"
  AI валидирует: "1 м² листа на дверь — маловато, обычно 3 м²"
       ↓
Этап 4: Настройка процессов (песочница)
  Клиент двигает блоки процессов визуально
  "Тестовый прогон" — видит результат
  Переставил → прогон → "Так лучше"
  Доволен → кнопка "Применить" → боевая система
       ↓
Этап 5: Застрял? → Колл-центр (бесплатно)
  Кнопка "Позвонить" — живой человек
  Оператор видит экран клиента
  Видит карточку: история, статус настройки
  Помогает довести до конца
       ↓
Этап 6: Работает
  За 2 дня — система настроена
  Работники уже работают
       ↓
Этап 7: Доработки (платная услуга)
  Нужна кастомизация → удалённо, по запросу
```

---

## Колл-центр

### Карточка клиента

Оператор видит полную картину при звонке:

- **Компания** — название, отрасль, контакт
- **История звонков** — дата, длительность, саммари, результат
- **Speech-to-text** — все разговоры транскрибируются и сохраняются
- **Саммари** — AI генерирует краткое содержание каждого звонка
- **Статус системы** — какие модули настроены, что осталось
- **Контекст** — оператору не нужно переспрашивать, всё в карточке

### Речевая аналитика

- Транскрипция всех звонков (speech-to-text)
- AI-саммари каждого звонка с результатом
- Автоматическое обновление карточки клиента
- Аналитика: на чём застревают → улучшаем AI-онбординг → нагрузка на колл-центр падает

### Принцип

Колл-центр — не расход, а инвестиция в продукт. Каждый звонок = данные о том, где продукт недоработан. Со временем AI справляется с всё большей долей вопросов, колл-центр нужен всё меньше.

| Уровень | Что входит | Стоимость |
|---|---|---|
| **Старт** | AI-онбординг, базовая настройка, колл-центр (2 дня) | Бесплатно |
| **Подписка** | Система работает, обновления, поддержка | N ₽/мес |

Ценообразование по подписке — определить после пилота (ориентир: 3 000–15 000 ₽/мес в зависимости от размера).

## AI-движок: что он умеет

### Импорт данных
- **Excel** — парсит любую структуру, понимает кривые таблицы, объединённые ячейки, комментарии
- **Голос** — speech-to-text, понимает производственный сленг
- **Текст** — свободный ввод, AI структурирует в номенклатуру
- **Фото** — распознавание товаров, ценников, накладных

### Экспертиза по отраслям
- Знает типовые BOM (металлические двери, окна, мебель и т.д.)
- Знает нормы расхода материалов
- Подсказывает забытые компоненты
- Валидирует спецификации (расход, нормативы)
- Предлагает шаблоны от похожих производств

### Обучение на данных клиентов
- Каждый клиент = данные о реальных процессах
- Типовые BOM по отраслям пополняются
- Нормы расхода уточняются
- Паттерны ошибок накапливаются
- Следующему клиенту — ещё проще и точнее

## Песочница процессов

### Концепция
Визуальный конструктор, где клиент настраивает производственные процессы перетаскиванием блоков. Каждый блок — этап (раскрой, сварка, покраска, сборка). Стрелки — последовательность.

### Тестовый прогон
Клиент нажимает "Тестовый прогон" — система симулирует производственный цикл:
- Показывает загрузку участков
- Показывает узкие места
- Показывает время цикла
- Предупреждает о проблемах

### Итерации
Переставил блоки → прогон → увидел результат → переставил ещё → прогон. Когда доволен — кнопка "Применить" переносит конфигурацию в боевую систему.

---

## Данные — конкурентный барьер

```
Клиент 1 (двери)     → типовой BOM дверей, нормы расхода металла
Клиент 2 (мебель)    → типовой BOM мебели, нормы расхода ДСП
Клиент 3 (окна)      → типовой BOM окон, нормы расхода профиля
...
Клиент N             → AI знает N отраслей, M тысяч BOM

Конкурент начинает с нуля → нет данных → AI не может помогать
Мы → N клиентов данных → AI помогает лучше → больше клиентов
                        = flywheel / network effect
```

## Метрики успеха

### Продукт
- Время от регистрации до работающей системы < 2 дней
- % клиентов, настроившихся без колл-центра > 90%
- Retention через 3 месяца > 80%

### Колл-центр
- Среднее время звонка < 10 мин
- % вопросов, которые AI мог бы решить (→ улучшаем AI)
- Снижение нагрузки с каждым месяцем

### Бизнес
- CAC (стоимость привлечения клиента) — стремится к 0 (продукт продаёт себя)
- LTV/CAC > 5
- MRR (monthly recurring revenue) — рост месяц к месяцу

## Риски

| Риск | Митигация |
|---|---|
| AI плохо парсит кривые Excel | Fallback на ручной ввод, улучшаем промпты |
| Клиенты не хотят облако | Возможность on-premise (Docker) |
| Сложно объяснить ценность | Демо-режим: "попробуйте на своих данных за 10 минут" |
| Колл-центр не масштабируется | AI берёт на себя всё больше → колл-центр для сложных случаев |
| Конкуренты скопируют | Данные клиентов = барьер, который нельзя скопировать |
